# F4 v0.2 Trial Dataset（試金石データ）

本ディレクトリは、
**F4 v0.2 における試験結果を
「判断可能な試金石データ」として提供するための成果物一式**
である。

---

## 1. このデータの目的

- HTML / Markdown という RAG 入力差分が
  Qommons.AI の応答に与える影響を
  **誤解なく観測・比較する**
- 他プロジェクトが
  **採否判断の根拠として再利用できる**
  データを提供する

本データは、
**結論を示すものではない**。

---

## 2. データ構成

本データは、以下の三層で構成される。

| 層 | 内容 |
|---|---|
| Raw | モデルの生回答（一次証拠） |
| Context | 実行条件・前提 |
| Summary | 人間向け差分整理 |

---

## 3. 利用上の注意

- summary のみを見て判断しないこと
- raw データを正本とすること
- SKIP（Inconclusive）は失敗ではない

### manifest.json の位置づけについて（重要）

本ディレクトリに含まれる `manifest.json` は、

- 本データセットに含まれる case・形式・ファイル構成を示す
  **目録（インデックス）**
- 人間およびツールが
  データセット全体を把握するための **補助メタデータ**

として提供される。

`manifest.json` は、

- Raw / Context / Summary の各 JSON データとは異なり、
- **JSON Schema Draft 2020-12 による validate の対象とはしない**

これは、manifest の責務を
**「構造検証」ではなく「構成把握・説明責任」に限定する**
設計上の判断である。

---

### 検証専用公式サンプルについて（重要）

本ディレクトリ配下の `samples/` に含まれる JSON ファイルは、

- JSON Schema Draft 2020-12 による validate 実証のための
  **公式サンプルデータ**
- スキーマ構造・必須項目・最小構成を示す目的で提供される

ものであり、

- 評価結果の正当性
- HTML / Markdown の優劣
- RAG 手法の有効性

を示すものではありません。

本プロジェクトは、判断材料となる **試金石データ構造の提供**に責務を限定します。
