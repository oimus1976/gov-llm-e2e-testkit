# 📘 F8-Set-1 実務手順 v1.0（FIX）

## 0. 目的（再定義しない）

本手順は、

> **HTML→Markdown 変換を行った条例データを用いて、
> Private Knowledge 上で QA を実行し、
> その結果を「価値判断のための材料」として記録・提供する**

ための **実務手順**である。

- 価値判断は行わない
- 評価・比較・結論は書かない
- 成果物仕様は変更しない

---

## 1. 前提条件（すべて満たしていること）

- reiki-rag-converter により
  **Markdown 化された条例データ一式（18本）が存在する**
- 元となる HTML 条例データは
  **すでにフォルダに保存されている**
- 成果物フォーマットは
  **成果物インターフェース定義 v0.1r+ を唯一の正本とする**
- QA 実行は
  **自動質問・回答収集スクリプトを必ず使用する**
- UI からの手動送信は **一切行わない**

---

## 2. 本セット（F8-Set-1）の定義

### 本セットの範囲

- **条例：1本**
- **質問：18問（固定セット）**
- **成果物：18ログ（1問＝1ファイル）**

※ 324ログは将来の拡張目標であり、本手順の対象外。

---

## 3. 入力データの扱い（FIX）

### 使用するデータ

- **Markdown 版条例のみを使用**

### 使用しないデータ（温存）

- HTML 版条例は
  - フォルダ上に存在していてよい
  - **本セット中は一切使用しない**

### 明示ルール

- HTML を Private Knowledge に投入しない
- HTML を `knowledge.files` に記載しない
- HTML を参照・比較した場合、
  **当該セットは無効とする**

---

## 4. 実行手段（最重要ルール）

### 必須

- 自動質問・回答収集スクリプトを使用する
- 全 18 問を
  **同一スクリプト・同一実行経路**で実行する

### 禁止

- UI からの手動送信
- 質問文のコピー＆ペースト
- 実行途中での手段切替
- 人的介入による再送・修正

---

## 5. 実行フロー（条例1本）

1. Markdown 条例データを Private Knowledge に投入
2. 自動質問スクリプトを起動
3. 質問 Q01 を送信
4. 表示された回答を取得
5. 成果物インターフェース定義 v0.1r+ に従い記録
6. Q02〜Q18 まで同様に実行
7. 18 問すべて完了後、実行を終了

---

## 6. 成果物の保存形式（再掲）

### ディレクトリ例

```text
f8_runs/
└─ YYYYMMDD/
   └─ ordinance_id/
      └─ Q01/
         └─ <model>_<context>.md
```

### 記載ルール

- `knowledge.files` には
  **実際に投入した Markdown ファイルのみ**を記載
- 推測・補完は禁止
- 引用がなくても `## Citations` セクションは必ず残す

---

## 7. ゴール条件（厳格）

### 完了条件

- 18 問すべての実行が成功している
- 18 ログすべてが v0.1r+ 準拠
- 欠損・再実行・中断がない

### 明示裁定

> **18ログ程度で中断が発生するようであれば、
> 本プロジェクトに次はない。**

---

## 8. 中断・失敗時の扱い

- 本セット中に中断・失敗が発生した場合
  → **当該セットは未完了として無効**
- 再実行する場合は
  - 日付を変え
  - 新しいセットとして実施する
- 既存ログは保持し、上書きしない

---

## 9. 本セット完了後の対応

- 成果物一式を
  **例規HTML変換プロジェクトへ「判断材料」として引き渡す**
- gov-llm-e2e-testkit 側では
  **結論・感想・評価を一切記載しない**

---

## 10. 禁止事項まとめ（再掲）

- 回答の良否評価
- HTML との比較
- Markdown 変換仕様の変更
- 成果物形式の独自拡張
- 判断・結論の記述

---

## 本手順の位置づけ

- 本手順は **実務手順であり設計書ではない**
- 手順変更が必要な場合は
  **新バージョンとしてのみ更新する**
- 本文書は **F8-Set-1 の唯一の正本**である
